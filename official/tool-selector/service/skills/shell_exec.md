# 执行命令 领域知识

## 可用工具
| 工具 | 功能 | 备注 |
|------|------|------|
| `shell_exec` | 执行终端命令/脚本（编译、安装依赖、运行测试、文件批处理等） | 在宿主 shell 中运行；返回 stdout/stderr |

## 工具选择指南
- 需要跑构建/测试/安装、或批量文件操作、或调用命令行工具 → `shell_exec`。
- 只读单个文件 → 用 `fs_read`，不要 `shell_exec` 调 cat。
- 只写/改文件内容 → 用 `fs_write`，不要 echo 重定向。
- 找文件/搜内容 → 用专用的列目录/搜索工具，比 shell 的 find/grep 更可控。
- Git 操作 → 优先用 `git_*` 专用工具，语义更清晰、更安全。

## 执行纪律
- **优先非破坏性操作**：读状态、跑测试、查信息这类只读命令可放手执行。
- **破坏性命令先确认**：删多个文件/目录、`rm -rf`、drop 数据库、改生产配置、批量覆盖等不可逆操作，执行前向用户说明影响并取得确认。
- **命令要可移植**：注意当前 shell 类型（PowerShell vs bash 语法不同），路径含空格要正确引号包裹。
- **拼接用户输入要转义**：避免命令注入，用户提供的值要正确引用。
- 安装依赖用固定/锁定版本，不用开放区间；可疑包名（疑似仿冒）先向用户核实。

## 常见陷阱
- 用 `shell_exec` 替代专用工具（cat/echo/find/grep）会绕过更安全的路径，且对用户不可见——能用专用工具就别用 shell。
- 交互式命令会挂起：避免需要人工输入的命令（交互式 rebase、需确认的提示），用非交互参数。
- 破坏性操作不可逆：执行 `rm -rf`、强制覆盖前务必确认目标路径正确、且已获用户同意。
- 长输出会刷屏：预期大量输出时加过滤或重定向，避免淹没上下文。
