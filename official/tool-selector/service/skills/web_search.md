# 网络搜索 领域知识

`web_search` 是 agent 内置的联网搜索工具，用于获取实时/最新信息。

## 可用工具
| 工具 | 功能 | 备注 |
|------|------|------|
| `web_search` | 按关键词搜索互联网，返回标题+摘要+URL | 必填 `query`；可选 `limit`(1-8，默认5) |

## 工具选择指南
- 查最新新闻/天气/股价/赛事/版本 → `web_search`
- 需要找某资源的 URL → `web_search` 先定位
- 已知具体 URL 要看内容 → 改用 `web_fetch`（不要用搜索）
- 项目内代码/文档问题 → 优先用文件工具，不要联网搜

## 技能协作流程
- 搜索 → 抓取：`web_search` 返回的是摘要，需要全文时拿 URL 再调 `web_fetch`。
- 搜索 → 保存：搜到的信息需要留存时，用 `fs_write` 写成本地文件。
- 逐步收窄：先用宽泛关键词搜，根据结果调整关键词再搜一次，比一次堆砌长 query 更有效。

## 常见陷阱
- `query` 必填，且尽量精确；堆砌过多无关词会稀释相关性。
- 英文关键词通常召回更好；查时效信息加年份/“latest”等限定词。
- `limit` 默认 5，别盲目调大——结果越多越占上下文，按需取。
- 搜索结果仅摘要，不要把摘要当全文引用；要细节务必 `web_fetch`。
- 不要搜索用户已经在对话里给出的信息，浪费一轮工具调用。
