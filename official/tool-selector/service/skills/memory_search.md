# 记忆存储与检索 领域知识

原生记忆工具，用于存取 AI 助手的长期记忆条目。与知识图谱记忆（`memory.md`，面向 `create_entities`/`relations` 等 MCP 工具）是两套独立机制；本文档面向 `memory_store` 和 `memory_search`。

## 可用工具
| 工具 | 功能 | 备注 |
|------|------|------|
| `memory_search` | 按关键词检索已存储的记忆 | 找回之前记住的信息，避免重复询问用户 |
| `memory_store` | 将一条信息写入长期记忆 | 用于持久化对用户/项目/上下文的关键事实 |

## 工具选择指南
- 重新开始对话/不确定之前记过什么 → 先 `memory_search`，命中就直接用，别再问用户。
- 用户提到"我之前说过""你应该记得" → `memory_search` 找回，而不是承认遗忘。
- 用户明确说"记住这个"、或发现值得跨会话保留的事实 → `memory_store`。
- 纯当前会话临时信息 → 不存不搜，直接用上下文。

## 检索原则
1. **关键词要贴近存储时的措辞**：记忆按关键词匹配，用与可能存储内容相近的词检索（存"深色主题偏好"，搜"主题"或"深色"而非"UI 喜好"）。
2. **先搜后写**：写入新记忆前先搜同类条目，有则更新/跳过，避免重复膨胀。
3. **命中即用**：搜到的信息直接采纳，不要搜到了还去问用户同样的问题。
4. **搜不到再问**：`memory_search` 无结果时，才向用户询问或按默认推进。

## 写入原则（配合 memory_store）
- 每条只含一个独立事实；用第三人称；用绝对日期（不用"今天/上周"）；有长期复用价值才存。

## 常见陷阱
- 不搜就问：用户最反感"我明明告诉过你"——存过的信息要先搜回来。
- 关键词错位导致搜不到：存与搜的用词要一致，否则记了等于没记。
- 相对时间词在跨会话记忆里失真，存储时务必用绝对日期。
- 注意与知识图谱（`memory.md`）分工：简单事实用本工具，实体间结构化关系用知识图谱 MCP。
