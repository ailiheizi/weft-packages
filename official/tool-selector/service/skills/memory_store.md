# 记忆存储与检索 领域知识

原生记忆工具，用于存取 AI 助手的长期记忆条目。与知识图谱记忆（`memory.md`，面向 `create_entities`/`relations` 等 MCP 工具）是两套独立机制；本文档面向 `memory_store` 和 `memory_search`。

## 可用工具
| 工具 | 功能 | 备注 |
|------|------|------|
| `memory_store` | 将一条信息写入长期记忆 | 用于持久化对用户/项目/上下文的关键事实 |
| `memory_search` | 按关键词检索已存储的记忆 | 找回之前记住的信息，避免重复询问用户 |

## 工具选择指南
- 用户明确说"记住这个"、或你发现一条值得跨会话保留的事实 → `memory_store`。
- 重新开始对话/忘记之前说过什么 → 先 `memory_search` 查是否已记录，再决定要不要问用户。
- 纯粹的当前会话临时信息（中间计算结果、一次性指令）→ 不要存。记忆是长期的，别当草稿纸。
- 关系型知识（实体之间有向关联）→ 用知识图谱（`create_entities` / `create_relations`），不用原生记忆。

## 写入原则
1. **先搜再写**：存储前先 `memory_search` 确认是否已有同类条目；有则更新/跳过，避免重复。
2. **每条只含一个独立事实**：不要把多个不相关信息塞进一条——混合条目搜索时难命中。
3. **用第三人称描述**：写"用户偏好深色主题"而非"你喜欢深色主题"或"我喜欢深色主题"，保持检索时的清晰性。
4. **用绝对日期**：写"2026-06-27"而非"今天""明天""上周"——记忆跨会话读取时相对时间无意义。
5. **有意义才存**：存可复用的事实（偏好、约束、项目上下文、反复被问到的信息），不存一次性指令或推导过程。

## 常见陷阱
- 不搜就存是最常见的错误：反复存储相同信息导致记忆膨胀、检索变慢、同一事实多版本冲突。
- 相对时间词（今天/刚才/下周）在跨会话记忆里会变成错误信息——必须用绝对日期。
- 存了不代表下次会用到：只有被 `memory_search` 命中的信息才有价值，写入时选好关键词让未来的搜索能找到。
- 注意与知识图谱的分工：简单事实（用户偏好/项目约束/单点信息）用本工具；实体间有结构化关系（"A 依赖 B""X 属于 Y"）用知识图谱 MCP。
